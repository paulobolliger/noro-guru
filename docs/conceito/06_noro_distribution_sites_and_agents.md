# 06 — NORO Distribution, Sites & Agents

> Documento estratégico sobre a camada de distribuição da NORO: sites, vitrines, landing pages, agentes, consultores, creators, líderes de grupo e canais comerciais distribuídos.

---

## 1. Definição

A camada de distribuição da NORO é o conjunto de recursos que permite que diferentes pessoas e empresas criem canais de venda de viagens usando a infraestrutura da plataforma.

Ela inclui:

- sites gratuitos e pagos;
- vitrines de agências;
- páginas de consultores;
- páginas de creators;
- páginas de viagens em grupo;
- landing pages de campanha;
- páginas de destino;
- páginas de produto;
- links de checkout;
- links de indicação;
- captação de leads;
- atribuição de origem;
- comissionamento;
- CRM;
- automações;
- conteúdo.

A tese central:

> Sites grátis não são brinde. São canais de distribuição.

---

## 2. Por que distribuição é central

A NORO não deve depender apenas de venda direta da plataforma para agências.

Ela deve permitir que cada usuário vire um ponto de distribuição.

```txt
agência
consultor
creator
líder de grupo
afiliado
fornecedor
Nomade Guru
```

Cada um desses perfis pode gerar demanda em sua própria rede.

A NORO deve transformar essa demanda em:

```txt
lead
→ proposta
→ pagamento
→ operação
→ comissão
→ recompra
```

## 4. Tipos de sites e páginas

### 4.1 Site da agência

Para tenants/agências formais.

Deve conter:

- marca da agência;
- serviços;
- destinos;
- pacotes;
- formulário de orçamento;
- WhatsApp;
- equipe;
- depoimentos;
- blog;
- páginas de viagem;
- checkout;
- área do cliente, futuramente.

### 4.2 Site do consultor

Para consultores independentes.

Deve conter:

- foto/perfil;
- especialidades;
- destinos;
- tipos de viagem;
- depoimentos;
- formulário de briefing;
- viagens em destaque;
- link para proposta;
- botão de contato;
- possibilidade de checkout.

### 4.3 Página de líder de grupo

Para viagens específicas.

Deve conter:

- nome da viagem;
- destino;
- datas;
- descrição;
- preço por pessoa;
- inclusões;
- exclusões;
- condições;
- prazo de pagamento;
- vagas;
- botão de interesse;
- pagamento individual;
- status do grupo, se aplicável;
- contato/suporte.

### 4.4 Página de creator

Para monetização de audiência.

Deve conter:

- identidade do creator;
- campanhas;
- viagens recomendadas;
- links de produto;
- tracking;
- formulário de interesse;
- checkout;
- conteúdos relacionados.

### 4.5 Página de afiliado

Mais simples.

Deve conter:

- link rastreável;
- campanha;
- produto;
- CTA;
- formulário ou checkout.

### 4.6 Página de destino

Para SEO e inbound.

Deve conter:

- guia do destino;
- melhores épocas;
- estilos de viagem;
- produtos relacionados;
- consultores especialistas;
- CTA para orçamento;
- FAQ;
- conteúdo gerado/assistido por IA e revisado.

### 4.7 Página de produto

Para itens vendáveis.

Exemplos:

- seguro viagem;
- visto;
- tour;
- transfer;
- hotel;
- pacote;
- cruzeiro;
- experiência;
- grupo.

### 4.8 Página de cotas/presente de viagem

Para lua de mel, aniversário, formatura ou presente coletivo.

Deve conter:

- história da viagem;
- meta;
- cotas;
- contribuições;
- mensagem para presente;
- pagamento;
- status;
- agradecimento.

## 5. A lógica do site grátis

O site grátis é uma ferramenta de aquisição e ativação.

### 5.1 Para a NORO

Ajuda a:

- atrair usuários;
- reduzir barreira de entrada;
- gerar distribuição orgânica;
- criar inventário de páginas;
- aumentar leads;
- vincular usuários ao ecossistema;
- justificar upgrades;
- alimentar CRM;
- impulsionar checkout.

### 5.2 Para o usuário

Ajuda a:

- parecer profissional;
- divulgar sua marca;
- captar leads;
- vender sem estrutura própria;
- compartilhar ofertas;
- organizar grupos;
- ter presença digital rápida.

### 5.3 Cuidado

Site grátis sem limite pode gerar custo e baixa qualidade.

É preciso definir:

- limites;
- templates;
- subdomínio;
- marca NORO;
- recursos pagos;
- padrões mínimos;
- moderação, se necessário.

## 6. Modelo de domínio e subdomínio

### 6.1 Modelo inicial

Usar subdomínios ou slugs.

Exemplos:

```txt
noro.guru/nomade
noro.guru/paulo
noro.guru/viagem-miami-zezinho
sites.noro.guru/nomade
nomade.noro.guru
```

### 6.2 Modelo pago

Permitir domínio próprio.

Exemplos:

```txt
www.agenciaexemplo.com.br
viagens.consultor.com.br
disney.comunidade.com.br
```

### 6.3 Recomendação

Começar com slug/subdomínio controlado.

Domínio próprio entra em plano pago.

## 7. Templates

Templates devem ser simples, bonitos e focados em conversão.

### 7.1 Templates iniciais

- agência geral;
- consultor especialista;
- viagem em grupo;
- creator;
- destino;
- produto;
- cotas de viagem;
- campanha promocional.

### 7.2 Estrutura comum

Todo template deve ter:

- headline clara;
- prova de confiança;
- descrição;
- CTA;
- formulário;
- contato;
- possibilidade de checkout;
- rastreamento.

### 7.3 Não exagerar no design

A prioridade é:

```txt

captar lead
vender
medir conversão
```

Não é ganhar prêmio de design com animação que pesa mais que mala de cruzeiro.

## 10. Jornada de distribuição

### 10.1 Fluxo básico

```txt
usuário cria página
→ compartilha link
→ lead entra
→ CRM registra origem
→ proposta é criada
→ cliente paga
→ comissão é calculada
→ venda é operada
```

### 10.2 Fluxo com grupo

```txt
líder cria viagem
→ página do grupo é gerada
→ participantes se cadastram
→ cada um paga individualmente
→ sistema acompanha status
→ operação confirma reservas
→ líder acompanha comissão
```

### 10.3 Fluxo com creator

```txt
creator escolhe campanha
→ personaliza página/link
→ divulga para audiência
→ leads/vendas são rastreados
→ comissão é calculada
```

## 11. Distribuição e comissionamento

A camada de distribuição precisa conversar diretamente com o motor de comissão.

### 11.1 Origem comercial

Toda venda deve ter campos como:

```txt
source_type
source_id
tenant_id
campaign_id
agent_id
leader_id
creator_id
affiliate_id
```

### 11.2 Regras de comissão

A comissão pode depender de:

- origem do lead;
- quem atendeu;
- quem fechou;
- quem operou;
- tipo de produto;
- plano do tenant;
- margem disponível;
- campanha.

### 11.3 Conflitos de atribuição

É preciso prever conflitos.

Exemplo:

```txt
cliente veio por link do creator
mas foi atendido por consultor da agência
e comprou produto de fornecedor global NORO
```

A plataforma deve definir regras claras.

No Brasil, a distribuição passa por WhatsApp.

A NORO deve aceitar isso como realidade.

### 13.1 Funcionalidades importantes

- botão de WhatsApp;
- mensagem pronta;
- link rastreável;
- envio de proposta;
- lembrete de pagamento;
- notificação de status;
- resumo de atendimento;
- integração futura com WhatsApp Business API.

### 13.2 Cuidado

WhatsApp não pode virar buraco negro operacional.

Tudo que for importante deve voltar para o CRM:

- lead;
- proposta;
- pagamento;
- documentos;
- status;
- suporte.

## 20. O que não fazer

### 20.1 Não dar site solto sem CRM

Site precisa alimentar funil.

### 20.2 Não criar marketplace público cedo demais

Rede sem governança vira feira livre com bug.

### 20.3 Não prometer renda fácil

A proposta deve ser profissionalização.

### 20.4 Não deixar usuário configurar tudo sozinho

Templates e fluxos guiados são essenciais.

### 20.5 Não separar conteúdo de venda

Conteúdo deve levar a lead, proposta ou checkout.

## 23. Relação com arquitetura multi-tenant

A distribuição precisa respeitar tenant e permissões.

### 23.1 Cada página deve saber

- a quem pertence;
- quem pode editar;
- quem recebe lead;
- quem recebe comissão;
- qual checkout usa;
- qual marca aparece;
- quais produtos pode vender.

### 23.2 White-label

Algumas páginas podem mostrar:

- marca do tenant;
- powered by NORO;
- marca NORO oculta em plano white-label.

### 23.3 Sub-redes

No futuro, um tenant pode ter sua própria rede de consultores/líderes.

---

## 25. Conclusão

A distribuição é uma das partes mais importantes da NORO.

O produto não deve apenas organizar vendas que já existem.

Ele deve permitir a criação de novos canais de venda.

A tese final:

> A NORO transforma sites, páginas, links, consultores, creators e líderes de grupo em canais conectados de venda de viagens.

A plataforma vence se conseguir fazer uma coisa muito bem:

```txt
qualquer pessoa ou empresa com demanda potencial
→ publica uma oferta
→ capta interessados
→ vende com segurança
→ recebe comissão
→ opera com suporte
```

Esse é o motor de distribuição.
